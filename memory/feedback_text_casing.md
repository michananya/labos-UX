---
name: Text casing rule
description: UI text must use sentence case — only first word capitalized, rest lowercase
type: feedback
---

Always use **sentence case** for all UI text: labels, button text, dialog titles, placeholders, column headers, chip labels, tooltips, and any other visible string.

- Correct: `Patient definitions`, `Print order form`, `Collection site`, `Advanced search`
- Wrong: `Patient Definitions`, `Print Order Form`, `Collection Site`, `Advanced Search`

**Why:** Consistent sentence case is the design system standard for LabOS.

**How to apply:** Every time you write a UI string — button text, title, placeholder, label, menu item, column header — capitalize only the first letter of the first word. Proper nouns (e.g. "LabOS") keep their own casing.
